# The Default Events

Kingshot Event Tracker starts with seven built-in event templates. These are meant to save setup time and give you sensible starting rules for the most common Kingshot events.

> Screenshot placeholder: event-bear-trap.png

Every default template follows one important rule:

- it can be edited
- it can be customized into a scoped copy
- it cannot be deleted

That protection is part of the platform safety rules. See [Safety Rules You'll Run Into](../roles/protection-rules.md).

## The seven default templates

| Event | Main style | Notes |
|---|---|---|
| [Bear Trap](bear-trap.md) | single-session score event | Damage-based alliance rally |
| [Viking Vengeance](viking-vengeance.md) | single-session score event | Points-based alliance defense |
| [Alliance Mobilization](alliance-mobilization.md) | multi-day cumulative event | Daily totals build toward one running score |
| [Alliance Brawl](alliance-brawl.md) | multi-stage event | Stage choice on import matters |
| [Kingdom of Power](kingdom-of-power.md) | kingdom-wide multi-stage event | Usually tracked with kingdom results and KvK prep |
| [Strongest Governor](strongest-governor.md) | multi-stage individual event | Daily stage or total standings |
| [Sanctuary Battle](sanctuary-battle.md) | participation and control event | Not a normal score event |

## What all seven have in common

All default events already come with:

- a category and visual style
- import aliases to help screenshot matching
- reward tracking turned on
- participation tracking turned on

Most of them are ordinary score events. Sanctuary Battle is the exception and should be treated differently.

## Why editing is allowed

The default templates are not meant to lock you into one exact setup forever. You can adjust them to fit how your kingdom or alliance wants to track:

- score labels
- stage labels
- import rules
- participant policies
- analytics visibility

If you need a variant that only your kingdom or alliance should use, create a custom copy instead of forcing everyone onto one shared version.
