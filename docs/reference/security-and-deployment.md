# Privacy and Safety Safeguards

This page describes user-facing safety expectations without disclosing sensitive service details.

## Account and session safety

- The platform uses protected sign-in and session handling.
- Administrator access is restricted by role and scope.
- The platform applies browser safety protections to reduce common web risks.

## Private uploads

Screenshots and import assets are private. A signed-in user can view an asset only when their role and current kingdom/alliance scope allow it. Supreme Admins retain platform-wide access for support and administration.

Private files are not intended to be shared outside the platform. Deleted files are unavailable until an authorized restore workflow completes.

## Spreadsheet safety

Spreadsheet uploads are treated as untrusted input. The platform accepts supported formats only and rejects oversized, malformed, unsafe, or out-of-scope uploads. Formula-like content and external references are not processed as executable content.

## Accessibility safeguards

Icon-only controls have accessible labels. Dialogs use clear dialog behavior, manage keyboard focus, support Escape where appropriate, and return focus to the invoking control. Responsive layouts and readable contrast remain part of the ongoing quality baseline.

## What users and administrators should do

- Keep accounts and personal processor keys private.
- Review screenshots and imports before accepting data.
- Give people only the role and scope they need.
- Report suspected account or privacy issues to a platform administrator.
