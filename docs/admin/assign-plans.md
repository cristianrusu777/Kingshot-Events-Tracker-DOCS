# Assign Plans Directly

This guide is for `Supreme Admin` users only.

Direct assignment is the fast admin path for putting a plan on a kingdom or alliance from the subscriptions dashboard.

## Use with care

Direct assignment **bypasses the normal request and payment workflow**.

That makes it useful when:

- payment was already handled outside the app
- you are setting up a new kingdom from scratch
- you are correcting an admin-side subscription state

It is not the normal customer-facing path. The normal path is still:

1. [Request a Subscription](../subscriptions/request-subscription.md)
2. [Payment Instructions & Completing a Request](../subscriptions/payment-instructions.md)
3. [Handle Subscription Requests](subscription-requests.md)

## How it works

From the [Subscriptions & Usage Manager](subscriptions-dashboard.md):

1. Open the relevant kingdom or alliance row.
2. Choose **Assign**.
3. Select a compatible plan.
4. Confirm the assignment.

The modal only offers plans compatible with that subject type.

## When to prefer this over the request flow

Use direct assignment when the subscription decision is already settled and you do not need a message-thread record of the request.

If you need a clearer approval trail, stay in the request workflow instead.
