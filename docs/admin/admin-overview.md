# Platform Administration Overview

This guide is for `Supreme Admin` users only.

It is the map of the admin area, not the full detail for every page. Use it as the entry point into the platform-level tools.

## What lives in the admin area

The admin area is where you manage the platform across all kingdoms.

The most important parts are:

- [Subscriptions & Usage Manager](subscriptions-dashboard.md)
- [Create & Edit Plans](manage-plans.md)
- [Assign Plans Directly](assign-plans.md)
- [Suspend & Unsuspend](suspend-subjects.md)
- [Handle Subscription Requests](subscription-requests.md)
- [Configure Support Links](support-links.md)
- [Email Settings](email-settings.md)
- [Theme Settings](theme-settings.md)
- [Registration Settings](registration-settings.md)

You also have the broader senior-admin tools already documented elsewhere:

- [Create & Manage Kingdoms](../how-to/manage-kingdoms.md)
- [Create a User](../how-to/create-user.md)
- [Assign or Remove Roles](../how-to/assign-roles.md)
- [Read the Audit Logs](../reference/audit-logs.md)

## First-day setup checklist

If you are setting up a fresh platform:

1. Create the first [kingdom](../how-to/manage-kingdoms.md).
2. Create the first admin [users](../how-to/create-user.md).
3. Review or create [plans](manage-plans.md).
4. Configure [support links](support-links.md).
5. Configure [email settings](email-settings.md).
6. Review [registration settings](registration-settings.md).
7. Open the [subscriptions dashboard](subscriptions-dashboard.md) and make sure the platform state looks correct.
