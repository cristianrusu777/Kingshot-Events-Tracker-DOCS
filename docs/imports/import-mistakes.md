# Fixing Import Mistakes

This page is the calm decision guide for "something about my import is wrong."

## If one row is wrong

Edit the row on the review page.

Use this when:

- the name is slightly wrong
- the score needs correction
- the wrong player was matched

See [Review Import Rows](review-rows.md).

## If one row should not be used at all

Ignore the row.

Use this when:

- the screenshot included junk text
- the same player was detected twice
- the row is clearly not part of the event you are saving

## If the import found the wrong player

Try **Rematch** first.

If that still looks wrong, edit the row or add the missing player before accepting.

## If the whole OCR pass looks bad

Upload the screenshot again with a better provider or clearer image.

Today, that is the practical replacement for an in-place reprocess workflow.

## If you already accepted and it was wrong

Use [Delete an Import & Roll Back Its Changes](delete-and-rollback.md) when you need to undo the tracked applied changes.

Then, if useful:

- restore the import as a draft
- re-review it
- re-accept only the correct rows

## If the screenshot context was wrong

Do not force the rows through. Start again with the correct:

- kingdom
- alliance
- event
- date
- stage or total mode
