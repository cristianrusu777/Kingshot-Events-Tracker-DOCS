# Work Inside an Instance

An instance detail page is the working area for one event run. This is where you review the session setup, participant snapshot, source batches, and saved result rows.

> Screenshot placeholder: instance-detail.png

## What you can see at the top

The summary table shows:

- kingdom and alliance
- date range
- current stage and total stages
- duration mode and score-entry mode
- scoring behavior
- lock state
- number of source batches

## Participant snapshot

For normal events, the instance page shows the event snapshot or eligible-player list. This is where you can see:

- the event alliance
- the player's current alliance
- the eligibility status
- join and leave snapshot dates
- review warnings

If a status is unclear, use [Participant Eligibility Statuses](../reference/participant-eligibility.md).

## Stage dates

For multi-stage events, the page can show a **Stage Dates** section. Senior editors can update those dates unless the instance is locked.

## Session rows and source batches

Lower on the page, you can review:

- **Source Batches** that fed the instance
- **Session Rows** for the saved results themselves

That makes the page useful both for "where did this come from?" and for direct row cleanup.

## Special case: Kingdom of Power

KvK instances show extra snapshot information and a link into the separate KvK Prep workflow. Use [Run a KvK Prep Session](kvk-prep.md) for before/after power capture.

## Locked instances

If the event is locked, editing and importing are blocked until the instance is unlocked. See [Locked Instances & Unlock Requests](session-lock.md).
