# Records & Record Batches

A **record batch** is a group of saved result rows tied to one event and date. It gives you a clean way to inspect what was saved together.

## Where record batches come from

Batches can come from:

- manual input
- screenshot imports
- spreadsheet imports

They are useful when you want to trace saved rows back to one entry action.

## What the list shows

The **Record Batches** page can show:

- date
- event
- kingdom
- alliance
- input source
- status
- row count
- last updated time

From the list, you can open:

- the batch detail
- the related event results
- the related import, when there is one

## What you can do inside a batch

On a batch detail page, authorized users can:

- review all saved rows together
- edit score, status, position, review flags, and notes
- add a row
- delete a row

This is the best page when you care about one saved batch rather than one whole instance.
