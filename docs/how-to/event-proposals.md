# Propose & Review Template Changes

The tracker includes an **Event Updates** workflow for proposed template changes. This helps when someone wants a change reviewed before it becomes the live event setup.

> Screenshot placeholder: event-proposals.png

## What this page is for

Use it when:

- someone proposed a change to a template
- you need to compare what would change before accepting it
- you want a simple accept or refuse decision

## What reviewers can see

The page can show:

- pending proposals
- the template being changed
- field-by-field before and after values, when available
- the proposal status

## Review actions

For each proposal, reviewers can:

- **Accept**
- **Refuse**

The backend also supports a review note, so teams can keep a short reason for the decision.

## Good practice

- review the changed fields, not just the event name
- pay extra attention to scoring mode, stage behavior, and participant policy changes
- refuse and ask for a cleaner proposal if the change bundles too many unrelated edits together
