# Browse & Filter Players

The **Players** page is the main working list for player records. It helps you find people quickly, narrow the list to the right scope, and open the next action you need.

> Screenshot placeholder: players-list.png

## What this page is for

Use the directory to:

- switch kingdom or alliance context
- search by player name
- filter by status, attribute, reward state, or API state
- sort the table
- open a player profile
- kick, edit, sync, delete, or restore a player when your role allows it

## Start with the context section

The **Context** area is important because it controls which players you are looking at.

You can usually narrow by:

- kingdom
- alliance
- search text
- status
- attribute
- reward state
- API state

Admins can also use **Show deleted** to include deleted players in the list.

## Read the table

The **Player Directory** table shows a compact view of:

- player name
- current scope
- profile ID link state
- API sync state
- level
- alliance role
- status
- attributes
- participation
- results count

## Common actions from the list

- Open the player profile.
- Sync the Kingshot profile for a linked player.
- Kick the player from the current alliance.
- Edit the player.
- Delete the player.
- Restore a deleted player.

What you see depends on your role and your current scope.

## Helpful filters

- Use **API linked** and **local only** when checking sync coverage.
- Use **reward eligible** when reviewing reward-readiness.
- Use the attribute filter when you are tracking a manual or analytics-driven segment.
