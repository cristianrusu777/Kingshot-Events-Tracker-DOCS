# Manage Nickname History

Nickname history helps the tracker understand that old imports and old event records may refer to the same person under different names.

> Screenshot placeholder: player-nicknames.png

## Why this matters

Without nickname history, you can end up wondering why:

- an older import used a different name
- a player seems to disappear between events
- a manual review row did not match the current visible name

Nickname history keeps that story together.

## Open the nickname area

1. Open the player's profile.
2. Go to **Nickname History**.

## Add a nickname

1. Enter the former or alternate nickname.
2. Select **Add nickname**.

Use this for old in-game names, common shorthand, or spelling changes that matter for matching.

## Make a nickname current

Use **Set current** when the old nickname has become the real current player name again.

When you do that:

- the player's current visible name changes
- the history stays intact

## Remove a nickname

Use **Remove** only for a nickname that should not stay in history.

The current nickname cannot be removed until another nickname is made current first.

## Good practice

- Keep meaningful past names instead of cleaning them away too aggressively.
- Update nickname history before rechecking messy imports.
- If the person's identity changed along with alliance membership, review both the nickname and the alliance history together.
