# Manual Attributes & Status Overrides

Manual attributes are for human judgment that should sit on top of normal player tracking. Status overrides are the stronger version: they tell the app which status to treat as the player's main one until you clear it.

## Open the tools

1. Open the player's profile.
2. Go to **Player Attributes**.

## Assign a manual attribute

1. Choose an attribute from **Assign manual attribute**.
2. Add a short reason if useful.
3. Select **Assign**.

This is a good fit when you want to record a decision that analytics alone should not guess.

## Save a status override

Use the override controls when the player's main status should be forced to a specific value for now.

You can:

- choose the override status
- add an override reason
- save the override
- clear the override later

## What is the difference

- A manual attribute adds information.
- A status override changes which status the tracker should treat as primary.

That means an override is more direct and should be used more carefully.

## Good practice

- Leave a short reason whenever another admin may need to understand the decision later.
- Clear overrides when they are no longer true.
- Recheck the player's graphs and recent records if the manual decision affects how you read their activity.
