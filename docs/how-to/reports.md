# File & Track Reports

Reports are the app's lightweight issue and follow-up system.

Use a report when something needs attention, discussion, or review.

## What you can do with reports

- create a report
- comment on a report
- review its status
- resolve it
- reject it

## When to use a report

Good examples:

- a data issue that needs review
- a question about a result or player record
- a follow-up item for leadership

If the specific problem is "this deleted thing should come back," also read [Request & Approve Restores](restore-requests.md).

## Typical flow

1. Create the report.
2. Add comments if more context is needed.
3. A senior role reviews it.
4. The report is resolved or rejected.
