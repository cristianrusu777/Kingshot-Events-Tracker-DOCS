# Create a Custom Event

Create a custom event when none of the default templates fits exactly, or when your kingdom or alliance needs its own variation of an existing template.

## When to create one

A custom event is a good idea when:

- you track an event the app does not seed by default
- you want different labels, stage rules, or visibility than the shared default
- you want a kingdom-only or alliance-only version of a template

If a default template is already close, you may prefer to customize or edit it instead. See [Edit an Event Template](edit-event-template.md).

## Basic steps

1. Open **Event Templates**.
2. Make sure your current context is correct.
3. Enter the event name.
4. Select **Add Event Template**.
5. Open the new template and finish its settings.

## Scope matters

The app creates custom templates in the scope you are working in:

- `Supreme Admin` without a selected kingdom creates a global template
- a kingdom context creates a kingdom template
- an alliance context creates an alliance template

If you meant to create a scoped template, check your context first.

## After creation

A new custom event starts simple. After that, you usually need to set:

- duration style
- score entry mode
- score labels
- analytics visibility
- participant policy settings

Use [Configure Event Settings](event-settings.md) for the detailed settings page.

## Good practice

- create the template first
- configure the rules second
- create real instances only after the template behaves the way you want
