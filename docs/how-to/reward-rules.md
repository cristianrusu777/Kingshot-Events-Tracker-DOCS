# Set Up Reward Rules

**Reward rules** are the configuration step behind the rewards system. They tell the app what counts as reward-worthy performance.

This guide is for `Supreme Admin`, `King`, and `Alliance Leader` users who are allowed to manage reward settings. If you only need to review the final output, go to [Review Reward Eligibility](rewards.md).

## What reward rules do

A reward rule looks at saved analytics and decides who should be marked as eligible for a reward. That can be based on things like:

- minimum points
- minimum participation percentage
- which events count
- how many missed events are still acceptable
- the time period being checked

This is the setup step. The app then uses those rules to build the output you read on the [Rewards](rewards.md) page.

## Open the page

Open **Settings** and then **Rewards**.

Depending on your role, you may be able to create rules at kingdom scope, alliance scope, or both.

## What you can set

The rules page includes fields such as:

- rule key and label
- minimum points
- minimum participation percentage
- included events
- maximum missed events
- period
- icon, color, and priority
- whether the rule is enabled

There is also a **Preview** area so you can check what a rule is meant to represent before relying on it.

## Scope matters

Reward rules can be tied to:

- a whole kingdom
- one alliance inside that kingdom

Set the scope carefully. A kingdom rule affects a wider audience than an alliance rule.

## Recalculate after changes

After changing rules, use the available recalculation action if the rewards output does not refresh right away. This helps the app rebuild eligibility from the updated rule set.

## Good practice

- Keep rule names clear so leaders can understand them on the rewards page.
- Start simple before creating many overlapping rules.
- Review [reward eligibility](../getting-started/glossary.md#reward-eligibility) after major rule changes.
- Treat this page as the rule setup step and [Rewards](rewards.md) as the review-and-handle step.
