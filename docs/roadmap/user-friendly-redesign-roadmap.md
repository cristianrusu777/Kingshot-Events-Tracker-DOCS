# User-Friendly Redesign Roadmap

> **Planned / Future · Not implemented yet · Primarily Free**

## Planned improvements

- role-based home pages that emphasize the next useful action
- guided kingdom/alliance setup
- event and instance creation wizards
- an import review assistant that explains confidence, conflicts, and consequences
- compact controls for dense tables
- clearer subscription, grant, locked, and suspended states
- useful empty states with setup links
- plain-language settings with examples and tooltips
- reward and cleanup assistants
- consistent success, warning, and error summaries

## Interaction principles

1. Users should know what to click next.
2. Dangerous actions should show affected records first.
3. Imports should always distinguish draft rows from saved results.
4. Premium locks should explain what is missing and how access could be obtained.
5. Tables should preserve identity, status, and primary actions at every size.
6. Recommendations should show evidence, not just a verdict.

## Tier direction

- **Free:** navigation, accessibility, safe actions, setup, clearer review, and better empty states.
- **Premium Both:** saved layouts, richer analytics navigation, recommendation workflows, and report composition.
