# Notifications & Discord Roadmap

> **Planned / Future · Not implemented yet · Free in-app notices, Premium Both external automation**

## Potential notifications

- import processing completed or failed
- review queue needs attention
- event reminder window approaching
- reward candidates ready for review
- quota warning or suspension risk
- unlock, restore, grant, or subscription request updated
- scheduled report ready

## Discord ideas

- approved weekly alliance digest
- approved kingdom event-health summary
- event reminder with no private player details by default
- reward-review reminder linking back to the app
- import-quality alert for editors

## Controls required before release

- opt-in per notification type
- channel and recipient allowlist
- preview before first send
- role/scope permission check at send time
- quiet hours, rate limits, deduplication, and retry limits
- redaction templates and minimal-data defaults
- audit record for configuration and deliveries

The system must never post player-level performance, reward, or behavioral information to a shared channel without explicit authorized configuration.
