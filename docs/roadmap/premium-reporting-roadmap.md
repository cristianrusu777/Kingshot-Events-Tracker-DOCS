# Premium Reporting Roadmap

> **Planned / Future · Not implemented yet · Premium Alliance, Premium Kingdom, and Premium Both**

## Planned report families

| Future report | Primary scope | Planned contents | Intended tier |
|---|---|---|---|
| Weekly alliance report | Alliance | attendance, contribution, risks, rewards, review debt | Premium Alliance |
| Monthly kingdom report | Kingdom | alliance comparisons, kingdom event health, grants, coverage | Premium Kingdom |
| Player report card | Player/alliance | attendance, consistency, event strengths, reward history | Premium Alliance |
| Event report | Alliance or kingdom | participation, scores, stages, change from comparison event | Premium Both |
| Reward eligibility report | Alliance | rule evidence, candidates, handled/pending state | Premium Alliance |
| Import quality report | Alliance or kingdom | confidence, conflicts, corrections, unresolved rows | Premium Both |
| Subscription usage report | Alliance or kingdom | usage, warnings, cleanup candidates, forecast ranges | Premium Both |
| Inactive-player report | Alliance | evidence-based inactivity and data caveats | Premium Alliance |

## Planned output formats

- PDF for a polished, fixed summary
- CSV for simple data exchange
- Excel for multi-sheet analysis and follow-up
- Discord summary for an approved, privacy-safe digest
- dashboard card for an in-app snapshot

## Premium reporting features

- one-click weekly alliance briefing
- one-click reward-candidate list
- event performance comparison
- player activity report cards
- saved report definitions
- scheduled generation with explicit recipients
- branded cover, date window, scope, and data-coverage note
- drill-through links back to the source view

## Trust requirements

Every report should state its scope, generation time, filters, metric definitions, missing-data coverage, and whether values are accepted, corrected, or estimated. Predictions must be labeled as estimates and separated from historical facts.

Discord or email delivery must be opt-in, previewable, permission-aware, and limited to approved channels/recipients.
