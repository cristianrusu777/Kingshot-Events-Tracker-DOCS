# Responsive Mobile Web Roadmap

> **Planned / Future · Not implemented yet · Free foundation with Premium enhancements**

## Goal

Make the existing web product comfortable on phones and tablets without hiding essential context, review warnings, or destructive-action impact.

## Planned Free experience

- responsive sidebar and context switcher
- readable dashboards, forms, and usage panels
- stacked tables with pinned player/event identity
- compact filters that remain discoverable
- touch-friendly review actions and confirmation dialogs
- mobile screenshot upload with visible draft status
- accessible typography, focus states, and contrast

## Planned Premium enhancements

- mobile-friendly analytics drill-downs
- saved custom views
- report previews and share controls
- recommendation inbox with explainable evidence

## Design constraints

- Important columns must not disappear silently.
- “Draft,” “saved,” “locked,” and “suspended” states must remain obvious.
- Bulk actions must show scope and impact before confirmation.
- Desktop remains available for dense administration and complex review.

## Success criteria for a future release

A leader should be able to check alerts, upload a screenshot, review a small import, inspect reward candidates, and understand subscription status from a phone without horizontal page scrolling.
