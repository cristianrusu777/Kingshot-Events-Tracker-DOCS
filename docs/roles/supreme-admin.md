# Supreme Admin Guide

This is the **start here** guide for `Supreme Admin` users. It is a map of the platform from the top level, plus a first-day checklist. It does **not** repeat every detailed how-to page. Use it as your reading path.

## What only a Supreme Admin can do

Some actions are not just "high permission" actions. They are hard-gated to Supreme Admin specifically, even if a custom role has powerful permissions.

These Supreme-Admin-only areas include:

- the platform-wide subscriptions and usage manager
- plan creation and plan editing
- direct plan assignment
- suspension and unsuspension of kingdoms and alliances
- the admin subscription requests queue
- support links configuration
- role and permission management

If you need the full guardrail wording, see [Safety Rules You'll Run Into](protection-rules.md).

## First-day checklist

If you are setting up the platform from scratch, follow this order:

1. Read [Roles Explained](overview.md) and [Finding Your Way Around](../getting-started/navigating.md).
2. Create your first [kingdom](../how-to/manage-kingdoms.md).
3. Create the first [users](../how-to/create-user.md) and [assign roles](../how-to/assign-roles.md).
4. Review or create your [plans](../admin/manage-plans.md).
5. Configure [support links](../admin/support-links.md) so subscription requests have somewhere to point.
6. Enable and review [registration settings](../admin/registration-settings.md).
7. Configure [email settings](../admin/email-settings.md) and send a test message.
8. Review the [subscriptions dashboard](../admin/subscriptions-dashboard.md).

## Common limits even for you

Supreme Admin is the strongest role, but a few safety rules still apply:

- you cannot delete your own account
- you cannot remove the last active Supreme Admin
- some destructive actions are soft-delete first, not immediate purge

Those rules are explained in [Safety Rules You'll Run Into](protection-rules.md).
