# King Guide

This is the **start here** guide for `King` users. It is a kingdom-level reading path: what you run, what you cannot do, and where to go next for each job.

## What a King is responsible for

A King runs **one kingdom** and the alliances inside it. In practice that means:

- managing the kingdom and its alliances
- creating users and assigning roles inside that kingdom
- managing players, events, instances, imports, analytics, rewards, reports, and restore requests
- offering premium kingdom grants to alliances

For the broad role summary, start with [Roles Explained](overview.md).

## The most important rule to understand

Kings have an **ownership** rule around deletion.

A King:

- can only delete users they created
- can only delete alliances they created
- can **never** delete Supreme Admin users
- can **never** delete users who registered independently

The exact blocked messages are already documented in [Safety Rules You'll Run Into](protection-rules.md). Read that page early, because it explains the most confusing refusals Kings run into.

## Premium and grants

Kings manage kingdom premium sharing from the kingdom side.

Read these together:

- [Grant Premium to Your Alliances](../subscriptions/kingdom-grants.md)
- [Share Kingdom Quotas with Alliances](../subscriptions/allocations.md)

One important correction to remember: the number of alliances you can grant is **not** a fixed platform-wide number. It comes from the kingdom plan's current grant-slot limit.

## Things a King cannot do

A King cannot:

- manage the global subscriptions dashboard
- create or edit plans
- suspend kingdoms or alliances
- handle admin subscription requests
- configure support links
- manage platform roles and permission catalogs

Those are Supreme-Admin-only areas. See [Supreme Admin Guide](supreme-admin.md) if you need the platform-level map.
