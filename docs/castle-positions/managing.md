# Managing Castle Positions

1. Confirm the Kingdom.
2. Create or open the cycle; set title and application window.
3. Review active stages: date, UTC start time, availability, and resources.
4. Configure positions, slot duration/count, capacity, and public resource fields.
5. Open applications and share the public link.
6. Review, link, and decide on candidates.
7. Build and save each stage; resolve conflicts and unplaced candidates.
8. Finalize the reviewed stage.
9. Publish and check notification results.
10. Reopen, edit, finalize, and publish a new version if required.
11. Close the cycle when applications should stop; records can later be archived.

Do not represent a draft as final. A stage without a date, positions, or valid schedule requires setup before appointments can be promised.
