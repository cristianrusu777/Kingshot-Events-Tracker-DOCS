# Castle Positions roles and access

Castle Positions has its own Kingdom boundary. A role elsewhere in the tracker does not automatically give Castle Positions administration access.

| Role | Access | Limit |
|---|---|---|
| Player / applicant | Apply, choose availability, opt into email, view own application and appointments | Cannot review others or edit schedules |
| Minister of Justice | Configure a cycle, review applications, build, finalize, publish, reopen, and notify schedules | Only the assigned Kingdom; no unrestricted platform administration |
| King | The same operating access for their own Kingdom | Cannot manage another Kingdom |
| Supreme Admin | Can select and manage Castle Positions across Kingdoms | Use platform-wide access carefully |

**Read-only** means a user can see information but cannot change it. A user without permission receives an access message and should contact their King, Minister of Justice, or Supreme Admin.

See [Roles Explained](../roles/overview.md) for the broader role model.
